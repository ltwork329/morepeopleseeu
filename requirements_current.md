# 当前需求真相表

## 目标

做一个本地化短视频自动生产工具。用户把文案导入系统后，系统按批次和编号管理文案、音频、视频素材、字幕、标题、预览截图和最终导出文件。

## 已确认

1. 文案输入只先支持两种方式：复制粘贴、本地 Excel 上传。
2. 每次导入生成一个批次号。
3. 每条文案在批次内生成独立编号。
4. 标题和正文必须分开保存。
5. 标题字号和字幕字号必须分开控制。
6. 默认配音语言为粤语，后续可选普通话和英语。
7. 视频素材必须按库存状态管理：素材可用、素材残片、素材已用。
8. 自动剪辑按音频时长加 1 秒取素材，除非用户手动指定时长。
9. 每次使用素材后，必须扣减该素材剩余可用时长。
10. 标题和字幕都要支持调整大小、格式和位置，并支持鼠标拖动预览。
11. 导出时要生成成片，也要生成预览截图或证据图，方便判断效果。
12. 粤语字幕识别第一版使用 FunASR 命令行调用。
13. 所有界面和功能按钮必须人性化，清晰可见，避免复杂操作。
14. 工作自动化最后再做，先把单条链路每一步梳理清楚。
15. 预览时必须自动抓取准备使用的视频素材首帧，作为素材确认画面。
16. 已经使用过的视频素材绝对不复用。
17. 批次号和文案编号由系统自动生成，并在导入页和任务详情里清楚显示。
18. 标题必须和文案一起上传或导入，每一个视频都必须有一个标题。
19. 页面流程必须按用户真实顺序展示：文案上传、文案生成音频、音频结合视频生成最终文件。
20. 声音克隆和文案生成音频必须有清楚入口，不能藏在任务详情里。
21. 文案编号必须带日期，例如 `20260426_0001`，方便区分哪天生成。
22. 文案上传页不放标题字号、字幕字号或样式设置。
23. 标题和字幕的颜色、大小、摆放位置必须在视频合成页调整。
24. 视频合成页要先抓取准备使用的视频首帧，再在首帧上调整标题和字幕位置。
25. 标题和字幕位置既要支持滑杆调整，也要支持鼠标拖动。
26. 整个项目围绕抖音/短视频成片输出，默认竖屏 9:16。
27. 标题和字幕默认按短视频规则单行展示，不能因为稍微调大就自动断成两行。
28. 素材不依赖文件名，用户只需要把素材放进固定文件夹。
29. 固定素材目录为：`local_materials/unused`、`local_materials/fragments`、`local_materials/used`。
30. 系统自动抓取素材时只从 `local_materials/unused` 获取。
31. 使用后的素材移动或登记到 `local_materials/used`，残片移动或登记到 `local_materials/fragments`。
32. 标题和字幕要有短视频默认样式方案，用户可以直接使用，不必每次从零调整。
33. 声音克隆完成一次后，后续任务可以跳过声音克隆，直接在生成音频页选择已有声音。
34. 生成音频页不需要复杂进度条，只需要选择声音、生成音频、显示结果。
35. 视频合成页必须显示合成进度条、当前任务状态和本页合成记录。
36. 合成记录要就近显示在视频合成页，不能只放在远处的导出记录页。
37. 视频合成页要让用户看到已完成多少条、处理中多少条、待处理多少条，以及当前这一条的状态。
38. 生成音频页必须先选择已上传的文案记录，再选择声音并生成音频。
39. 视频合成页必须先选择已生成音频的记录，再抓取素材首帧、调整标题和字幕。
40. 视频合成页的预览区必须始终清晰可见，没有任务时显示大号空预览框。
41. 素材库页面不需要手填素材名称、分类、秒数；这些应由扫描脚本从固定素材文件夹读取。
42. 素材库要显示未用、残片、已用三个库的文件数和总时长。
43. 文案上传页不选择语言，语言在“生成音频”步骤里选择。
44. 页面文案要极简，保留必要状态，去掉冗长提示语。
45. 导出记录不再单独分页，合并到视频合成页面展示。
46. 视频合成页必须包含：选择已生成音频记录、首帧预览、标题字幕样式调整、输出进度、历史记录。
47. 素材库时长展示优先按三类文件夹下“每条素材时长”显示，并保留分类汇总时长。
48. 输出记录要显示编号，便于快速核对。
49. 素材库支持一键初始化文件夹并立即扫描。

## 批次和编号

批次号用于一次导入任务，例如：

```text
batch_20260426_001
```

文案编号用于批次内单条内容，例如：

```text
batch_20260426_001_item_0001
batch_20260426_001_item_0002
```

每条文案的资产都挂在文案编号下面：

| 资产 | 说明 |
|---|---|
| 原始标题 | 用户输入或 Excel 导入的标题 |
| 原始正文 | 用户输入或 Excel 导入的正文 |
| 优化正文 | DeepSeek 处理后的正文 |
| 配音音频 | MiniMax 生成的音频 |
| 字幕文件 | 识别或生成的字幕 |
| 标题样式 | 标题字号、颜色、位置、字体等 |
| 字幕样式 | 字幕字号、颜色、位置、字体等 |
| 素材使用记录 | 使用哪个素材、起止时间、使用时长 |
| 预览截图 | 导出前后的效果截图 |
| 最终视频 | 输出 mp4 |

## Excel 导入字段

第一版 Excel 至少支持：

| 字段 | 必填 | 说明 |
|---|---|---|
| title | 是 | 视频标题 |
| body | 是 | 文案正文 |
| language | 否 | `yue`、`mandarin`、`english`，默认 `yue` |
| title_font_size | 否 | 标题字号 |
| subtitle_font_size | 否 | 字幕字号 |
| duration_seconds | 否 | 手动指定视频时长 |

如果 Excel 没有提供字号，使用系统默认值。

## 素材库存规则

每个视频素材进入系统后，要记录：

| 字段 | 说明 |
|---|---|
| material_id | 素材编号 |
| file_path | 本地文件路径 |
| category | 素材分类 |
| total_duration_seconds | 原始总时长 |
| available_segments | 仍可使用的时间段 |
| used_segments | 已使用时间段 |
| fragment_segments | 不适合继续自动使用的残片 |
| status | `available`、`fragment`、`used` |

硬规则：已经进入 `used_segments` 的视频片段绝对不能再次使用。素材被完整用完后进入素材已用，素材已用不参与任何后续匹配。

## 素材扣减例子

如果一个视频素材总时长是 120 秒，某条音频是 30 秒，自动剪辑需要 `30 + 1 = 31` 秒。

使用后：

```text
已用：0s - 31s
剩余可用：31s - 120s
剩余可用时长：89 秒，也就是 1 分 29 秒
```

如果剩余片段太短，不能满足下一条视频的最小时长，就进入素材残片。

## 素材状态定义

| 状态 | 定义 |
|---|---|
| 素材可用 | 至少有一个剩余片段能满足当前任务需要 |
| 素材残片 | 剩余片段存在，但太短或不适合自动剪辑 |
| 素材已用 | 没有可继续使用的片段 |

素材已用是终态。除非用户明确重置素材使用记录，否则系统不能自动把素材已用改回素材可用。

## 剪辑选择规则

第一版使用最简单稳定规则：

1. 优先选择同分类下剩余可用时长足够的素材。
2. 默认从最早可用片段开始截取。
3. 每次截取后立刻写入素材使用记录。
4. 如果指定时长，则按指定时长取素材。
5. 如果未指定时长，则按音频时长加 1 秒取素材。
6. 如果没有足够素材，任务直接失败并提示缺少哪个分类的素材。
7. 已使用片段绝对不参与下一次匹配。
8. 素材已用状态的文件绝对不参与下一次匹配。

## 暂定默认值

| 项 | 默认值 |
|---|---|
| 语言 | 粤语 |
| 视频比例 | 9:16 |
| 自动视频时长 | 音频时长 + 1 秒 |
| 标题字号 | 64 |
| 字幕字号 | 42 |
| 残片阈值 | 小于 5 秒进入残片 |
| FunASR 调用方式 | 命令行调用 |

## 待确认

1. Excel 表头是否可以固定为英文：`title`、`body`、`language`。
2. 素材分类是手动建分类，还是按文件夹名自动分类。
3. 残片阈值是否固定 5 秒。
4. 同一个素材是否允许跨批次继续使用。
5. 标题默认显示在顶部、中间还是底部。

## 字幕识别规则

1. 粤语字幕识别指定调用 FunASR。
2. FunASR 输出字幕后，系统保存为可编辑字幕轨道。
3. 字幕内容、字号、颜色、描边、位置都必须能在预览页调整。
4. 调整字幕位置时，预览框必须显示真实导出位置。
5. 普通话和英语字幕识别引擎后续可扩展，但第一版先保证粤语 FunASR 链路可用。

## FunASR 待落地事项

1. 本机安装方式已定：第一版使用命令行调用。
2. 确认粤语模型名称和模型缓存目录。
3. 确认输入音频格式，建议统一转为 `wav` 后再识别。
4. 确认输出格式，第一版统一转成 `srt` 和内部 JSON。
5. 如果 FunASR 识别失败，任务直接失败并显示错误，不做伪字幕。

## 人性化界面规则

1. 页面第一眼必须看得懂当前在做哪一步。
2. 主要按钮必须清晰可见，按钮文字直接说明结果，例如“导入文案”“生成音频”“预览视频”“导出成片”。
3. 不把多个复杂功能塞进一个按钮。
4. 每个任务要显示当前状态，例如“待配音”“已生成音频”“字幕待调整”“可导出”。
5. 需要用户处理的问题要直接说明原因，例如“这个素材剩余 4 秒，低于 5 秒残片阈值”。
6. 标题和字幕调整页要有真实预览框，拖动位置和最终导出位置一致。
7. 字号、颜色、位置等设置要放在预览旁边，用户调整后能立刻看到效果。
8. 批量自动化要保留“预览后导出”和“一键自动导出”两种入口。
9. 危险操作要清楚提示，例如删除素材、清空批次、重置素材使用记录。
10. 第一版不做花哨设计，优先保证清楚、稳定、省步骤。

## 单条链路优先规则

第一阶段不做完整自动化，先把单条任务跑清楚：

1. 导入一条文案。
2. 优化一条文案。
3. 生成一条音频。
4. 匹配一个视频素材。
5. 抓取准备使用素材的首帧。
6. 生成一条字幕。
7. 调整标题和字幕位置。
8. 预览一条视频。
9. 导出一条成片。

每一步都要能单独查看状态、结果和失败原因。

## 素材首帧规则

1. 系统选中准备使用的视频素材后，必须先抓取该素材的首帧。
2. 首帧显示在预览页，帮助用户确认当前准备使用的是哪个素材。
3. 如果实际截取不是从 0 秒开始，首帧应使用本次截取片段的开始时间画面。
4. 首帧要保存到该文案编号下面，方便后续查记录。
5. 导出记录页要显示首帧或最终预览截图。
