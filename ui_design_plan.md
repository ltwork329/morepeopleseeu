# 本地短视频工具页面设计方案

## 设计目标

做一个本地可用、清晰、省心的短视频生产工作台。整体气质是可爱、青春、文艺、潮流，但不能花哨到影响操作。

第一版不需要登录和改密码，因为只在本地使用。

## 视觉方向

| 项 | 设计 |
|---|---|
| 气质 | 可爱、青春、文艺、潮流 |
| 主色 | 奶油白、蜜桃粉、薄荷绿、天空蓝 |
| 强调色 | 明亮珊瑚色，用于主要按钮 |
| 字体感觉 | 圆润、亲切、清楚 |
| 卡片 | 轻柔阴影，圆角适中，不堆太多层 |
| 图标 | 使用清楚的小图标，辅助识别功能 |
| 动效 | 轻微过渡，只用于状态变化和进度提示 |

## 首页结构

首页就是工作台，不做宣传页。

| 区域 | 内容 |
|---|---|
| 顶部 | 工具名称、当前本地状态、MiniMax/FunASR/DeepSeek 配置状态 |
| 左侧 | 流程导航：文案导入、声音克隆、音频生成、素材库、字幕标题、导出记录 |
| 中间 | 当前批次任务列表 |
| 右侧 | 当前任务详情、进度、预览截图 |

## 首页核心按钮

| 按钮 | 位置 | 作用 |
|---|---|---|
| 导入文案 | 首页显眼位置 | 粘贴文案或上传 Excel |
| 新建批次 | 导入文案旁边 | 创建一个批次号 |
| 生成音频 | 任务详情区 | 调 MiniMax 生成配音 |
| 识别字幕 | 任务详情区 | 调 FunASR 生成字幕 |
| 预览视频 | 任务详情区 | 打开可视化调整页 |
| 导出成片 | 任务详情区 | 输出最终 mp4 |
| 一键自动完成 | 后期功能 | 自动跑完整批次，第一阶段先不做 |

按钮文字必须直接说明结果，不使用模糊词。

## 任务状态

每条文案任务必须清楚显示状态。

| 状态 | 含义 | 显示方式 |
|---|---|---|
| 待处理 | 已导入，还没开始 | 浅灰标签 |
| 处理中 | 正在执行某一步 | 蓝色标签 + 进度条 |
| 成功 | 当前流程完成 | 绿色标签 |
| 失败 | 某一步失败 | 红色标签 + 失败原因 |
| 待调整 | 需要人工调整标题或字幕 | 黄色标签 |

## 处理中进展

处理中不能只显示“处理中”，必须告诉用户正在做什么。

示例：

```text
正在优化文案  2/20
正在生成粤语音频  8/20
正在调用 FunASR 识别字幕  9/20
正在匹配素材  10/20
正在导出视频  12/20
```

每条任务详情也要显示当前步骤：

```text
当前步骤：FunASR 正在识别粤语字幕
已用时间：00:18
预计剩余：00:35
```

## 文案导入页

| 区域 | 功能 |
|---|---|
| 复制粘贴 | 粘贴标题和正文 |
| Excel 上传 | 上传本地 Excel |
| 批次号 | 自动生成，可手动改备注 |
| 字段预览 | 显示识别到的标题、正文、语言、字号 |
| 导入确认 | 确认后生成任务列表 |

Excel 导入后必须先预览，不直接进入自动流程。

## 素材库页

| 区域 | 功能 |
|---|---|
| 素材分类 | 按分类查看本地素材 |
| 素材可用 | 显示剩余可用时长 |
| 素材残片 | 显示不能自动使用的短片段 |
| 素材已用 | 显示已完全使用的素材 |
| 使用记录 | 显示被哪个批次、哪个编号使用 |

素材时长必须用用户能看懂的方式显示，例如：

```text
总时长 02:00
已用 00:31
剩余可用 01:29
```

素材库必须明显提示：已经使用过的视频素材绝对不复用。

素材已用区域只做查看，不提供自动继续使用入口。

## 预览调整页

| 区域 | 功能 |
|---|---|
| 左侧 | 真实视频预览和准备使用素材首帧 |
| 右侧 | 标题设置、字幕设置、导出设置 |
| 底部 | 时间轴、字幕列表 |

标题和字幕可以鼠标拖动。

拖动时要显示边框和中心线，让用户知道最终位置在哪里。

预览页必须自动展示准备使用的视频素材首帧。

如果本次截取片段不是从 0 秒开始，展示截取开始时间对应的画面。

首帧旁边显示：

```text
素材名称
素材分类
截取开始时间
计划使用时长
剩余可用时长
```

## 导出记录页

导出记录要清晰可看。

| 字段 | 说明 |
|---|---|
| 批次号 | 哪次导入 |
| 编号 | 哪条文案 |
| 标题 | 视频标题 |
| 状态 | 成功、失败、处理中、待处理 |
| 失败原因 | 如果失败，直接显示原因 |
| 输出文件 | 成片路径 |
| 预览截图 | 效果图 |
| 创建时间 | 任务创建时间 |
| 完成时间 | 导出完成时间 |

## 第一版页面顺序

1. 文案导入页
2. 批次任务列表
3. 素材库页
4. 预览调整页
5. 导出记录页

声音克隆和模型配置可以先放在设置页，不放到首页打扰主流程。

完整自动化最后做。第一阶段先把单条链路每一步做清楚，不急着做“一键全自动”。

## 下一步开发建议

第一步先做一个本地 Web 工作台骨架：

1. 左侧导航。
2. 批次任务列表。
3. 任务状态和进度条。
4. 文案复制粘贴导入。
5. Excel 上传导入。
6. 导出记录表。

这一步先不接 MiniMax、DeepSeek、FunASR，只把页面和本地数据结构跑通。

第二步再做单条链路：

1. 单条文案优化。
2. 单条音频生成。
3. 单条素材匹配。
4. 准备使用素材首帧抓取。
5. 单条字幕生成。
6. 单条预览调整。
7. 单条导出。

第三步再做批量和自动化。
