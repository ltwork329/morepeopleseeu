# 批量自动剪辑实施路线图（基础准备版）

## 目标定义（唯一）
在单项目内实现：**批量输入主题 -> 自动生成文案 -> 自动配音 -> 自动合成短视频 -> 自动质检 -> 批量导出**。

## Phase 0（第 1 周）：基础底座
| 模块 | 交付标准 |
|---|---|
| 仓库结构 | 完成目录规范与命名规范，避免跨项目混用 |
| 配置体系 | 环境变量、密钥、存储路径统一前缀 `svaf_` |
| 任务状态 | 统一状态机 + 错误码体系 |
| 最小台账 | `task_plan/progress/findings` 全量启用 |

## Phase 1（第 2-3 周）：MVP 全链路
| 模块 | 交付标准 |
|---|---|
| 文案引擎 | 模板化脚本生成，支持批量任务 |
| TTS 引擎 | 固定 2-3 套可控音色参数 |
| 自动剪辑 | 支持模板化镜头拼接、字幕烧录、BGM 混音 |
| 导出 | 支持至少 9:16 竖屏导出 |

## Phase 2（第 4-5 周）：批量稳定性
| 模块 | 交付标准 |
|---|---|
| 队列化执行 | 支持并行任务与重试 |
| 质检门禁 | 黑帧、静音、字幕越界、敏感词自动拦截 |
| 可观测性 | 任务耗时、失败率、失败原因看板 |
| 恢复能力 | 支持从中间步骤重跑，避免全链路重算 |

## Phase 3（第 6 周）：运营可用
| 模块 | 交付标准 |
|---|---|
| 任务工作台 | 批量新建、批量审核、批量导出 |
| 模板中心 | 文案模板、镜头模板、字幕样式模板 |
| 报表 | 日产能、通过率、平均生产时长 |

## 必须先定的技术决策（先不争论业务）
1. 渲染引擎：FFmpeg 管线还是 NLE 工程驱动（建议先 FFmpeg）
2. 队列系统：单机队列还是分布式队列（建议先单机可回放）
3. 素材池规范：命名、标签、版权来源、生命周期
4. TTS 提供方策略：单供应商先跑通，后续再抽象

## 验收口径（完成定义）
- 连续批量 100 条任务，端到端完成率 >= 95%
- 失败任务可按错误码重试，不需要人工改代码
- 出片时长、字幕、音频同步误差在可接受阈值内
- 每条任务可追溯输入参数与处理日志
