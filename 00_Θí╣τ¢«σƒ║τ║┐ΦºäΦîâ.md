# 短视频自动化项目基线规范（公司版）

## 结论先行
本项目唯一目标：**批量自动剪辑（文案 -> 音频 -> 视频）**。  
只做这一条业务链，不引入其他项目目标，不复用外部项目资料，除非你明确授权。

## 1. 项目边界（防撞规则）
- 项目代号：`sv-auto-factory`
- 统一命名前缀：`svaf_`（表名、队列名、环境变量、对象存储目录统一前缀）
- 目录隔离：仅允许读写当前仓库目录
- 资料隔离：不主动读取其他项目文件、历史方案、外部私有文档
- 需求隔离：不新增“直播、电商、CRM”等旁支需求

## 2. 业务闭环（必须完整）
1. 文案输入（主题、脚本模板、时长、平台）
2. 音频生成（TTS、语速、音色、停顿、重音）
3. 视频合成（素材匹配、字幕、BGM、转场、片头片尾）
4. 质检（画面时长、字幕同步、违规词、静音/黑帧检测）
5. 导出与分发（多平台规格、批量任务、失败重试）

## 3. 角色与权限（最小可用）
| 角色 | 核心权限 |
|---|---|
| 管理员 | 全部配置、账号、模板、发布渠道 |
| 运营 | 新建批量任务、模板选择、导出发布 |
| 审核 | 仅查看与审核、打回 |
| 观察者 | 只读报表 |

## 4. 核心数据对象（首批）
| 对象 | 关键字段 |
|---|---|
| project | id, name, platform, status |
| script_task | id, project_id, prompt, script, tags |
| audio_task | id, script_task_id, voice_id, speed, audio_url |
| video_task | id, audio_task_id, template_id, aspect_ratio, video_url |
| qc_report | id, video_task_id, score, issues, reviewer |
| publish_job | id, video_task_id, channel, publish_time, state |

## 5. 工作流状态机（必须落地）
- `draft -> queued -> processing -> review -> approved -> exported -> published`
- 失败统一进入：`failed`，必须带 `error_code + error_message`
- 禁止“静默失败”

## 6. 可靠性与恢复（必须）
- 对象存储：原始素材、音频、成片分目录存储
- 数据库每日备份；保留 7/30 天两档
- 任务可重放：同一输入参数可复现
- 失败任务可批量重试（按错误码筛选）

## 7. 测试与验收基线
- 单元测试：脚本生成、字幕切分、时间轴拼接
- 集成测试：文案->音频->视频全链路
- 回归测试：模板升级不影响既有任务
- 发布门槛：核心链路通过率 >= 98%，批量 100 条任务可稳定跑完

## 8. 交付物标准
- 每次变更必须同步更新：
  - `task_plan.md`
  - `progress.md`
  - `findings.md`
- 每次声称“完成”，必须附验证结果（命令/截图/样例输出）

## 9. 语音供应商规范（新增）
- 语音供应商支持：`OpenAI TTS` + `MiniMax 国际版 TTS`
- 当前新增要求：**MiniMax 国际版作为可选主供应商**
- 禁止在代码中硬编码密钥、租户信息、私有 endpoint
- 统一参数层：`text / voice / speed / emotion / format / sample_rate`
- 统一产物层：`audio_url / duration_ms / provider / model / request_id`
- 统一错误层：`provider_error_code / provider_error_message / retriable`
