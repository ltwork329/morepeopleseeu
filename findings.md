# findings

## 2026-04-26 新确认
1. 文案入口第一版限定为复制粘贴和本地 Excel 上传。
2. 所有导入内容必须按批次号和文案编号管理。
3. 视频素材按库存管理，状态分为素材可用、素材残片、素材已用。
4. 自动剪辑默认取音频时长 + 1 秒，并扣减素材剩余可用片段。
5. 粤语字幕识别调用 FunASR。
6. FunASR 输出结果必须转成可编辑字幕轨道，后续允许调整内容、字号、样式和位置。
7. FunASR 第一版采用命令行调用。
8. 所有界面和功能按钮必须人性化、清晰可见、避免复杂操作。

## 当前未决

2. 素材分类是否按文件夹名自动生成。
3. 素材残片阈值是否固定 5 秒。
4. 同一个素材是否允许跨批次继续扣减使用。
5. FunASR 粤语模型名称和模型缓存目录待确认。

## 已确认事实
1. 当前项目目录为空，尚无代码资产和依赖。
2. 你要求“只聚焦短视频从文案到音频到视频”，不碰其他项目资料。
3. 目标是“批量自动剪辑”，不是单条手工出片工具。
4. 已完成 6 个核心技能安装，可覆盖配音、转写、视频生成与流程验证。
5. 已补全网页可访问式技能栈，可直接进入 Web 工作台开发与上线准备。
6. 已将 MiniMax 国际版纳入语音供应商规范，并新增配置模板与校验脚本。

## 缺口与风险
1. 缺少 `hermes-agent` skill（规则中要求，但当前环境未发现）。
2. 缺少 `planning-with-files` 与 `gsd-method-guide`（已用文档流程替代）。
3. 当前无技术栈、无运行时、无任务队列与渲染实现，仍处于 0->1 准备阶段。
4. 部署平台虽都可用，但正式环境建议先锁定一个主平台，避免重复运维。
5. MiniMax 真实 API 调用尚未接入代码层，当前为规范与配置就绪状态。
